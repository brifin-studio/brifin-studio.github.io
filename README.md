## Роли

### Постановщик задачи
- Создаёт задачи через форму ТЗ
- Видит загруженность дизайнеров (вкладка "Загрузка дизайнеров")
- Видит статусы своих задач
- Комментирует задачи
- Редактирует задачи после отправки
- **НЕ двигает карточки по доске**
- **НЕ меняет статус задачи**

### Дизайнер
- Видит канбан-доску с 4 колонками
- Двигает карточки между колонками (drag-and-drop)
- Меняет статус задачи
- Редактирует все поля задачи
- Комментирует задачи
- Видит стикеры изменений на карточках
- Выбирает свой цвет при первом входе (6 вариантов)

---

## Авторизация
**Постановщик:** вводит имя (свободный ввод)
**Дизайнер:** выбирает из фиксированного списка + вводит имя + выбирает цвет при первом входе

Цвет дизайнера сохраняется в Firebase и используется для обводки его карточек и аватара.

## Канбан дизайнера

### 4 колонки с цветными полосами сверху:
| Колонка | Цвет полосы |
|---------|-------------|
| Новое | #4285f4 синий |
| В работе | #fbbc04 жёлтый |
| На правках | #ea4335 красный |
| Готово | #34a853 зелёный |

### Карточка на доске (свёрнутый вид):
- Строка 1: тип задачи (иконка + название) + приоритет-бейдж
- Строка 2: название задачи (до 2 строк, жирный)
- Разделитель
- Строка 3: аватар + имя дизайнера + дедлайн
- Просроченный дедлайн — красный цвет
- Обводка карточки = цвет дизайнера
- Лёгкий тонированный фон того же цвета (opacity 0.05)

### Приоритет-бейджи:
- Срочный → красный
- Высокий → оранжевый
- Обычный → зелёный

### Стикеры изменений (правый верхний угол карточки):
- 💬 + число = новые комментарии
- ✏️ + число = изменения полей
- 🔄 + число = смена статуса
- **Пропадают** когда пользователь открыл карточку и закрыл
- **Появляются снова** при новом изменении

### Модалка по клику на карточку:
**Шапка:** тип + приоритет + название + статус (меняет только дизайнер)
**Основной блок:** исполнитель / дедлайн / описание / формат
**Лог изменений:** автоматические записи "Артём изменил дедлайн: 30 июня → 15 июля"
**Комментарии:** лента с именем + датой, поле ввода (пишут оба)

### Права редактирования:
| Поле | Постановщик | Дизайнер |
|------|:-----------:|:--------:|
| Название | ✅ | ✅ |
| Описание | ✅ | ✅ |
| Дедлайн | ✅ | ✅ |
| Приоритет | ✅ | ✅ |
| Статус | ❌ | ✅ |
| Комментарии | ✅ | ✅ |

## Типы задач и формы ТЗ

### 🎬 Видео
- Платформа (pill-теги): YouTube / TikTok / Instagram / ВКонтакте / Rutube / Другое
- Ориентация: Горизонтальная 16:9 / Вертикальная 9:16 / Квадрат 1:1
- Разрешение (текстовое поле, например 1920x1080)
- Длительность (сек)
- Формат файла (pill-теги): MP4 / MOV / AVI / GIF / Другой
- Анимация Да/Нет (если Да — поле описания анимации)
- Кому / Дедлайн / Приоритет / Описание / Референсы

### 📱 Соцсети
- Название задачи + Платформа в одной строке
- Форматы (чекбоксы с размерами):
  - Сторис (1080×1920 px)
  - Пост (1080×1080 px)
  - Вертикальный пост (1080×1350 px)
  - GIF (до 15 сек)
- 9 карточек контента в сетке 3×3 (сворачиваемые)
  - Каждая карточка: текстовое поле + загрузка изображения
  - Поддержка вставки через Ctrl+V

### 📊 Презентация
**3 сценария (кнопки):** Создать новую / Внести правки / Адаптировать под новый объект

**Создать новую:**
- Редактор слайдов (белый прямоугольник 16:9)
- Кнопки "+ Текст" и "+ Изображение"
- Блок текста: теги типа (Заголовок / Основной текст / Список / Цифра/факт / Подпись)
- Блок изображения: загрузка файла + пометки-стикеры (жёлтые, перетаскиваемые) + полноэкранный режим
- Статус слайда: Готово к работе / Нужно обсудить
- Кнопка "+ Добавить слайд"

**Внести правки / Адаптировать:**
- Ссылка на оригинальный файл
- Редактор слайдов (такой же)
- Референс стиля + Дополнительные материалы

**Общие поля всех сценариев:** Кому / Дедлайн / Приоритет / Описание

### 🌐 Лендинг
**3 сценария:** Создать новый / Внести правки / Адаптировать

**Создать новый:**
- Блочный редактор секций ("Блок сайта")
- Типы блоков (pill-теги): Главный экран / Описание / Преимущества / Программа-Расписание / Цены / Форма заявки / Призыв к действию / Другое
- Внутри каждой секции: те же блоки Текст и Изображение
- Нумерация блоков крупным серым числом слева
- Синий левый бордер карточек

### 🏙️ Наружная реклама
**3 сценария:** Новый макет / Внести правки / Адаптация под форматы

**Новый макет:**
- Тип носителя (pill-теги, мультиселект): Билборд / Баннер на забор / Сити-лайт / Вывеска / Брандмауэр / Digital-экран / Другое
- Размер: ширина (мм) + высота (мм) + разрешение (dpi)
- Анимация Да/Нет
- Блочный редактор контента с тегами: Заголовок/УТП / Основной текст / Логотип / Фото/Рендер / Контакты / QR-код

**Внести правки:**
- Ссылка на файл + поле правок
- Загрузка макета с пометками-стикерами (как в презентации)

**Адаптация под форматы:**
- Загрузка оригинала + выбор новых форматов + размеры

Дизайн: оранжевый левый бордер (#f57c00), фон карточек #fff8f0

### 🖨️ Полиграфия
**4 категории (карточки):** Презентационный формат / Офисная полиграфия / Сувенирка / Навигация

**Типы по категориям (pill-теги):**
- Презентационный: Каталог / Буклет / Книга продаж / Флаер / Лифлет / Другое
- Офисная: Бланк / Паспорт объекта / Визитка / Тейбл-тент / Сертификат / Купон / Бейджи / Другое
- Сувенирка: Календарь / Кружка / Одежда / Пакет / Другое
- Навигация: Табличка / Дорхенгер / Генплан / Планировка / Другое

**Форматы зависят от категории:**
- Презентационный: А6 (105×148) / А5 (148×210) / А4 (210×297) / А3 (297×420) / Евро (99×210) / Свой размер
- Офисная: А5 / А4 / 90×50 мм (визитка) / Свой размер
- Сувенирка/Навигация: только поле "Свой размер (мм)"
- "Свой размер" → текстовое поле "Например: 60×40 мм"

**Тираж:** только для Презентационного / Офисной / Сувенирки
**Двусторонняя печать:** только для Презентационного / Офисной

## Общие технические детали

### Firebase
- Все задачи хранятся в Firestore
- Realtime обновления — все видят изменения без перезагрузки
- Test mode (открытый доступ на 30 дней, потом нужно обновить правила)

### Пометки на изображениях
- Жёлтые стикеры по клику на изображение
- Перетаскиваемые
- Текст с автопереносом (min 150px, max 280px)
- Полноэкранный режим с теми же пометками
- Синхронизация пометок между превью и полным экраном

### Редактирование задач
- Доступно обоим после отправки
- Открывается через кнопку в модалке карточки

### Вставка изображений
- Поддержка Ctrl+V в блоках изображений
